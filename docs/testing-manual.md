## 5. การจัดการงานซ่อม (Repairs)

### 5.1 สถานะงานซ่อม (Workflow)

```
pending (รอซ่อม)
    → waiting_parts (รออะไหล่)
    → quoted (เสนอราคา)
    → confirmed (ลูกค้ายืนยัน)
    → in_progress (กำลังซ่อม)
    → qc (ตรวจสอบ QC)
    → completed (ซ่อมเสร็จ)
    → delivered (ส่งคืนแล้ว)
    → cancelled (ยกเลิก)
    → claim (เคลม)
```

### TC-REP-001: สร้างงานซ่อมใหม่

| รายการ                | รายละเอียด                                                                                                                                                                                                                                                                                                                                                           |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ขั้นตอน**           | 1. ไปที่ งานซ่อม > สร้างใหม่ <br> 2. กรอกข้อมูลลูกค้า: ชื่อ, เบอร์โทร (จำเป็น) <br> 3. กรอกข้อมูลอุปกรณ์: ประเภท, ยี่ห้อ, รุ่น (จำเป็น) <br> 4. กรอก IMEI, Serial, สีเครื่อง, สภาพเครื่อง (ถ้ามี) <br> 5. กรอกอาการเสีย (จำเป็น) <br> 6. กรอกราคาประเมิน, วันส่งคืน <br> 7. กรอกเงินมัดจำ (ถ้ามี) <br> 8. เลือกระดับความเร่งด่วน: normal/urgent/vip <br> 9. กดบันทึก |
| **ผลลัพธ์ที่คาดหวัง** | - สร้างงานซ่อมสำเร็จ เลขที่ `REP-YYMMDD-XXXX` <br> - สถานะเริ่มต้น `pending` <br> - redirect ไปหน้าแสดงรายละเอียด                                                                                                                                                                                                                                                    |
